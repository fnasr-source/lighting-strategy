Appendix

Methodology & Data Sources

Data Collection Methodology

Instantly: Live API pull (2026-03-01) via campaign analytics and campaign detail endpoints. 2 SimLabs campaigns matched by name filter. Note: open_tracking=false, link_tracking=false on both campaigns.

Asana: Deep extraction via Task API including custom fields, comments, stories, and attachments for all tasks in projects 1212055797762548 and 1213388383279022. Links extracted and HTTP-validated.

Fathom: 758 total meetings filtered to 24 using strict criteria: title match (Simulations Labs / SimLabs / QYD) OR invitee domain (simulationslabs.com). Dedup key: recording_id.

Google Drive: Recursive mirror of client folder. 24 files catalogued. Large files (>50MB) preserved locally with hash verification.

Lead Unification: 70 Asana pipeline tasks parsed, deduplicated by email + company name, enriched with Instantly campaign context. 1 duplicate group collapsed → 69 final records.

Systeme.io: Tags and page 1 contacts fetched. Full pagination not implemented — data is incomplete.
