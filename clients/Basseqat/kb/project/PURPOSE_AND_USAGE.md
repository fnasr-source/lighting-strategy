# Purpose And Usage

## Why This Repo Exists
Admireworks needs one maintained starting system for new client projects.
Without it, the same strategic, operational, copy, and Firebase setup work gets repeated from scratch.

## What Success Looks Like
A new client repo should be able to start from this template and quickly move through:
1. source ingestion
2. research and market scanning
3. strategy synthesis
4. copy development
5. review and approval
6. implementation planning
7. admin and backend setup
8. build and operations

## What This Repo Must Preserve
- reusable process
- reusable doctrine
- reusable templates
- reusable prompts
- reusable structural contracts
- reusable Firebase verification and deployment discipline

## What Should Evolve Over Time
- stronger copy playbooks
- richer examples library
- stronger strategy review doctrine
- more reusable implementation code
- better seed scripts and setup automation
- tighter App Hosting and health runbooks
