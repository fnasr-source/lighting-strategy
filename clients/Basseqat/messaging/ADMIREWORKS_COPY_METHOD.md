# Admireworks Copy Method

This file turns StoryBrand and direct-response principles into a production workflow for client work.

Use it when moving from raw source material to approval-ready copy.

## Goal
Produce copy that is:
- grounded in source truth
- native to the market language
- commercially sharp
- supported by proof
- usable across pages, ads, email, WhatsApp, and sales flows

## Working Sequence
1. Extract the truth
2. Capture market language
3. Build the narrative spine
4. Define the offer logic
5. Map proof to claims
6. Map objections to responses
7. Choose the CTA ladder
8. Adapt by channel
9. Run the review rubric

## 1. Extract The Truth
Read briefs, transcripts, WhatsApp threads, sales notes, decks, offer docs, and approved pages.

Create five buckets:
- confirmed facts
- likely inferences
- unanswered questions
- proof assets already available
- direct quotes or phrases worth reusing

Output:
- a short fact sheet
- a gap list
- a language bank

## 2. Capture Market Language
Find the phrases real people use when they talk about:
- the pain
- the desired result
- failed alternatives
- why they hesitate
- what makes them trust

Sources can include:
- sales calls
- founder voice notes
- WhatsApp chats
- testimonials
- FAQs
- competitor reviews

Do not normalize the language too early.
Keep the rough edges that make it sound real.

## 3. Build The Narrative Spine
Use StoryBrand as the backbone:
- Character: who the asset is speaking to
- Problem: external, internal, and philosophical
- Guide: why this brand deserves trust
- Plan: what happens next
- CTA: what action should happen now
- Success: what life looks like after action
- Failure: what happens if the problem continues

Output:
- one clear narrative spine per audience
- one one-liner
- one simple articulation of the plan

## 4. Define The Offer Logic
Clarify:
- who the offer is for
- what it includes
- what result it promises
- why it is different
- what risk is removed
- what urgency is believable

If the offer is weak, unclear, or unsupported, flag it before writing heavy copy.
Good copy cannot rescue a vague offer.

Output:
- primary offer statement
- support offer or lead magnet if relevant
- CTA ladder by awareness stage

## 5. Map Proof To Claims
List the main claims the copy will make.
For each claim, attach the best available proof.

Example structure:
- claim
- proof type
- proof source
- strength: strong / moderate / weak
- note if client confirmation is still required

Claims without proof should be:
- narrowed
- rewritten
- or parked until proof exists

## 6. Map Objections To Responses
Build a practical objection ledger.

Minimum categories:
- trust
- cost
- time
- fit
- complexity
- fear of failure
- prior bad experience

For each objection, define:
- what the buyer is really worried about
- what proof or explanation reduces that worry
- where that objection should be handled in the journey

## 7. Choose The CTA Ladder
Do not use one CTA everywhere by habit.

Map CTA by stage:
- cold: learn more, see how it works, get the brief
- warm: book a call, request pricing, get the plan
- high intent: buy, enroll, start, confirm
- skeptical: start small, try first, see proof, talk to us

The CTA must fit both:
- awareness stage
- channel behavior

## 8. Adapt By Channel
Keep the strategic core stable, but change the delivery.

### Landing Page
- full argument
- proof and objection handling
- strongest guide and mechanism clarity

### Ad Copy
- one angle at a time
- fast tension
- dominant proof or belief shift

### WhatsApp
- conversational
- low-friction
- context-aware
- one action per message

### Email
- more space for logic, story, and proof
- strong subject line and first sentence

### Sales Call Or Discovery Prep
- questions first
- mirror buyer language
- use the same pain, proof, and CTA logic as the campaign

## 9. Review Before Approval
Use `COPY_REVIEW_RUBRIC.md`.

At minimum, reject copy that:
- invents certainty
- sounds generic
- lacks proof support
- uses non-native phrasing
- pushes a CTA too early

## Output Standard
Each major copy draft should include:
- audience
- awareness level
- angle or narrative spine
- main claim set
- proof set
- objections handled
- CTA direction
- open questions for the client

## What Good Looks Like
Good output feels like it came from a strategist and a copywriter working from real calls.

It should feel:
- specific
- calm where trust matters
- forceful where urgency is justified
- native to the buyer
- consistent across channels
