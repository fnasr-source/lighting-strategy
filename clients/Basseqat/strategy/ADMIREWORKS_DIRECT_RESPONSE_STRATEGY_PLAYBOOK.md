# Admireworks Direct Response Strategy Playbook

This is the canonical structure for strategy creation in client repos derived from this starter.

## Purpose
Create strategy that is:
- evidence-backed
- market-aware
- conversion-focused
- operational enough to build from

## Core Rule
Do not create strategy directly from a brief and one transcript if critical market context is still missing.
Complete the research layer first, then synthesize.

## Required Sequence
1. client KB intake
2. missing-inputs check
3. research layer
4. strategy synthesis
5. messaging and offer alignment
6. page and funnel specs
7. presentation build after written strategy approval

## Canonical Strategy Sections
1. Situation and objectives
2. Competitor analysis
3. Market gaps
4. Market opportunities
5. Target audiences and personas
6. Market and digital environment scanning
7. Messaging strategy
8. High value content offer
9. Funnel design
10. Social media strategy
11. Measurement and evaluation

## Anti-Generic Rules
- cite source files or research docs for meaningful claims
- do not import assumptions from another client project
- do not reuse old personas, offers, or channel advice unless revalidated
- if evidence is weak, mark it as inference
- if evidence is missing and important, ask before continuing

## Presentation Rule
After the written strategy is approved:
- use `strategy-kit/STRATEGY-SYSTEM.md`
- generate the visual presentation build
- keep presenter and client modes tied to approved strategy only
